# Agents Directory

**Purpose**: Agent constitution storage for HestAI consumers.

## Current Status

This directory contains the bundled core agent constitutions for the HestAI system.
These agents are available for binding via `mcp__hestai__bind`.

## Available Agents

### Core Architecture
- `technical-architect`: Complex system design and architectural decisions
- `design-architect`: High-level design patterns and structural planning
- `north-star-architect`: Vision alignment and long-term roadmap direction
- `principal-engineer`: Technical excellence and engineering standards
- `system-steward`: System-wide coherence and component integration
- `completion-architect`: Finishing phases and delivery assurance

### Implementation & Build
- `implementation-lead`: Build phase execution and task coordination
- `code-review-specialist`: Code quality enforcement and review
- `universal-test-engineer`: Test generation and coverage validation
- `error-architect`: Systematic error resolution and debugging
- `vibe-coder`: Rapid prototyping and experimental coding
- `wind-agent`: High-velocity implementation

### Validation & Quality
- `critical-engineer`: Production readiness and domain accountability
- `quality-observer`: Quality metrics and standards monitoring
- `validator`: General validation and verification
- `test-methodology-guardian`: Testing standards and protocol enforcement
- `security-specialist`: Security auditing and hardening

### Orchestration & Workflow
- `holistic-orchestrator`: Master workflow coordination
- `task-decomposer`: Breaking down complex tasks into actionable units
- `requirements-steward`: Requirements gathering and management
- `door-agent`: Entry protocols and context initialization
- `wall-agent`: Boundary enforcement and scope protection

### Specialist Domains
- `supabase-expert`: Supabase database and backend integration
- `smartsuite-expert`: SmartSuite platform integration
- `octave-specialist`: OCTAVE format compliance and documentation

## Usage

Consumers can use templates from this directory to create project-specific agent constitutions.

See `.hestai-sys/library/octave/octave-usage-guide.oct.md` for OCTAVE format reference.
